     - TMDB_API_KEY
     - KEYSTORE_BASE64 (from: keytool + base64)
     - KEYSTORE_PASSWORD
     - KEY_ALIAS (value: "movie_app")
     - KEY_PASSWORD

  3. Workflow builds automatically
  4. Download APK from Releases

ARCHITECTURE OVERVIEW
================================================================================

LAYER STRUCTURE:
  View (UI)           ← Activities, Fragments, Adapters
         ↓
  ViewModel           ← LiveData management, state preservation
         ↓
  Repository          ← Business logic, API + DB orchestration
         ↓
  DataSource          ← API (Retrofit) + Database (Room)

DATA FLOW:
  1. UI observes LiveData from ViewModel
  2. User action triggers ViewModel method
  3. ViewModel calls Repository method
  4. Repository queries API or Database
  5. Data returns via LiveData
  6. UI automatically updates

REACTIVE PROGRAMMING:
  ✓ Lifecycle-aware observers (no memory leaks)
  ✓ Automatic UI updates on data change
  ✓ Offline-first with local caching
  ✓ Smooth transitions between online/offline
